Stats panel гэж юу вэ?

Stats panel бол бодит ордер биш — энэ нь indicator дотор суусан backtest симулятор юм. Тэр нь lookback хугацаанд (default 12 сар) дараах логикийг candle бүр дээр давтан тоолж дүгнэдэг:

Entry (LONG/SHORT flip) → TP1 / TP2 / TP3 / TP4 шат → SL (trailing / BE / lock) → эсрэг flip дээр хаалт

Энэ нь та одоо харж буй тохиргоо (TP ×ATR, SL, BE-after-TP, ADX/HTF/Session филтер г.м) яг хэрхэн ажилласан байх байсныг тухайн график, тухайн timeframe дээр харуулна. Тохиргоо өөрчлөх бүрд тоонууд автоматаар шинэчлэгдэнэ.

⚠️ Анхаар: Эдгээр нь симуляцийн дүн (slippage, commission, spread ороогүй). Бодит арилжааны баталгаа биш, харин тохиргооны чанарыг харьцуулах хэрэгсэл.

  • Weak WR = тэдгээр trade-ийн winrate.
👉 Хэрэглээ: Weak % өндөр ба Weak WR нийт Win-ээс мэдэгдэхүйц доогуур бол Block weak-follow flips филтерийг асаа — winrate чирэгч муу entry-уудыг хасна. (Улаан өнгөтэй болоход анхаар.)

  • Fav WR = тэдгээр entry-ийн winrate.
👉 Хэрэглээ: Fav WR нь нийт Win-ээс 5%+ илүү байвал Premium/Discount filter-ийг асаа — counter-bounce trade-уудыг хасч winrate ахиулна. (Ногоон = сайн, улаан = таагүй байршилд их орсон.)

5. Алдагдлын мөр (Real Boost)

Losses — 4 -7.78%

4 = нийт алдагдалтай trade-ийн тоо. -7.78% = тэдгээрийн нийлбэр алдагдал.

Loss% / R — 28.6% 1.08R

  • Loss% = 28.6% — алдагдлын хувь (4/14). 50%-аас доош байх ёстой (улаан болвол алдагдал ялалтаас давсан гэсэн үг).
  • R (Expectancy Ratio) = 1.08R — нэг алдагдалд ногдох дундаж ашиг = дундаж win ÷ |дундаж loss|.

• R ≥ 1.0 → эерэг хүлээлттэй (дундаж ялалт ≥ дундаж алдагдал) → ногоон
• R < 1.0 → нэг ялалт нэг алдагдлыг нөхөж чадахгүй
1.08R нь эерэг талд: дунджаар 1 алдагдалд 1.08 ашиг ногдоно.

Хэрхэн "уншиж" дүгнэх вэ — товч checklist

Сайн тохиргооны шинж:

  1. Win% ≥ 55–60% — тогтвортой
  2. R ≥ 1.0 (Loss% / R мөр) — эерэг хүлээлттэй
  3. Net эерэг ба өсөн нэмэгдсэн
  4. Max DD хүлээж авах боломжтой (ж: < 10–15%, өөрийн эрсдэлийн хэмжээгээр)

  6. Weak / Fav мөрүүд улаан бол → холбогдох филтерийг асааж туршина
Хамгийн чухал хоёр тоо: Win% (тогтвортой эсэх) ба R (урт хугацаанд ашигтай эсэх). Энэ хоёр зэрэг сайн байж гэмээ системийг найдвартай гэж үзнэ.

KHAN | Sniper Pro — Stats Panel Guide

Explanation of every metric in the Stats panel (top-right corner of the chart) — 2026-06-17

What is the Stats Panel?

The Stats panel is not real orders — it is a backtest simulator built into the indicator. Over the lookback period (default 12 months) it replays the following logic on every candle:

Entry (LONG/SHORT flip) → TP1 / TP2 / TP3 / TP4 levels → SL (trailing / BE / lock) → close on counter-flip

It shows exactly how the current settings (TP ×ATR, SL, BE-after-TP, ADX/HTF/Session filters, etc.) would have performed on that specific chart and timeframe. Numbers update automatically whenever you change a setting.

⚠️ Note: These are simulation results (no slippage, commission, or spread included). Not a guarantee of real trading performance — use them to compare settings quality.

1. Core Performance

Trades — 14 (10/4)

Total closed trades. In parentheses (wins / losses): 14 total, 10 wins, 4 losses.

Win — 71.4%

Win rate = wins ÷ total = 10/14. The primary measure of system consistency.

Net — +13.29%

Net total return across all trades. How many percent this setup returned over the lookback period.

Avg — +0.95%

Average return per trade = Net ÷ Trades. Should be positive.

Max DD — -6.46%

Maximum Drawdown — deepest equity drawdown from peak. The primary risk metric. Lower is more stable (very large DD = high-volatility system).

Best / Worst — +8.25% / -3.01%

Best = largest gain on a single trade. Worst = largest loss on a single trade. A large gap between the two suggests results depend heavily on a few "lucky" trades.

2. Take Profit & Live Status

What percentage of opened trades reached each TP level:

  • 71% → reached TP1
  • 57% → reached TP2
  • 14% → reached TP3
  • 0% → reached TP4

Declining percentages toward higher TPs are normal. If TP3/TP4 reach is very low, consider lowering "Simulated full exit at" to TP2/TP1.

Now — +1.02%

Floating P&L of the currently open simulated position — move from entry to current price. Empty (--) when no position is open.

3. Risk / Active State

The trailing stop loss level of the current active position. Matches the red dashed line on the chart (KSniper: Trailing Stop Loss). Label changes to Breakeven / TP1 Lock when BE/TP1-lock has triggered.

  • 21.1 = current ADX value (trend strength)
  • 18 = minimum threshold required for entry
  • OK = ADX is above threshold, signal permitted. Below threshold → classified as chop (sideways market), entries blocked.

4. Quality Filters (Weak / Fav)

These diagnostic rows show which type of entry is dragging down or boosting the win rate, and advise which optional filter to enable.

Weak % — 0% (0)   /   Weak WR — --

  • Weak % = share of "weakly-followed flip" trades (count in parentheses). A weak flip means the flip candle's close direction mismatches the flip direction (e.g. LONG flip but candle closed red).
  • Weak WR = win rate of those trades.
👉 How to use: If Weak % is high and Weak WR is noticeably below overall Win%, enable the Block weak-follow flips filter — it removes low-quality entries that drag down the win rate. (Watch for the red coloring.)

Fav Loc — 21% (3)   /   Fav WR — 67%

  • Fav Loc = share of entries in the "favorable zone" of the ATR range (LONG → discount = lower half, SHORT → premium = upper half), count in parentheses.
  • Fav WR = win rate of those entries.
👉 How to use: If Fav WR is 5%+ higher than overall Win%, enable the Premium/Discount filter — removes counter-bounce entries and improves win rate. (Green = good, red = too many entries in unfavorable zones.)

5. Loss Row (Real Boost)

Losses — 4 -7.78%

4 = total number of losing trades. -7.78% = combined loss from those trades.

Loss% / R — 28.6% 1.08R

  • Loss% = 28.6% — loss rate (4/14). Should stay below 50% (red coloring means losses outnumber wins).
  • R (Expectancy Ratio) = 1.08R — average gain per loss = avg win ÷ |avg loss|.
👉 This is the key quality number for the system.
• R ≥ 1.0 → positive expectancy (avg win ≥ avg loss) → green
• R < 1.0 → a single win cannot cover a single loss
1.08R is positive: on average 1.08 wins' worth of profit per loss.

How to Read & Evaluate — Quick Checklist

Signs of a good setup:

  1. Win% ≥ 55–60% — consistent
  2. R ≥ 1.0 (Loss% / R row) — positive expectancy
  3. Net positive and growing
  4. Max DD acceptable (e.g. < 10–15%, based on your risk tolerance)
  5. Best/Worst balanced — not reliant on one "lucky" trade
  6. If Weak / Fav rows show red → enable the relevant filter and test
The two most important numbers: Win% (is it consistent?) and R (profitable long term?). Both must be solid before the system is considered reliable.
